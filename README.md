# Elite: Dangerous リアルタイムダッシュボード

## 概要
このアプリケーションは、戦闘、交易、探索、およびマテリアル収集などの統計情報をリアルタイムで表示するウェブベースのウィジェットです。
また、OBSと連携して録画開始時を起点としてタイムスタンプをイベントログとして記録する機能を持ちます。
各統計カードはドラッグ＆ドロップによって自由に並び替えることができ、レイアウトはリアルタイムで他のブラウザへ同期され、永続化されます。
さらに、主要な統計データ（賞金、利益、燃料残量）はグラフで視覚的に表示されます。

## 仕様
コンセプトは「今日の成果」であり、起動時当日のジャーナル及び、起動中に増えたジャーナルを収集対象とします。

日付を跨いだ場合も収集を継続して集計に加算し続けます。日付を跨いだあとに再起動した場合、前日の統計は集計対象外となるので統計がリセットされます。
ただし、ゲームのプレイイベント（ジャンプ、戦闘、取引など）がまだ発生していない場合に限り、ダッシュボード上の「RESUME SESSION」ボタンから直前のセッション状態を復元して再開することが可能です。

### 収集する統計情報
現在、以下の統計情報が収集・表示されます。

- **戦闘関連:**
  - 戦闘報酬の集計 (以下の総計)
    - 賞金額 (Bounty)
    - 戦闘貢献証券 (Combat Bond)
    - 1時間あたりの獲得賞金
    - 直近10キルの獲得賞金推移
  - ランク別撃墜数集計 (賞金首のみ)
  - 機種別撃墜数集計 (賞金首のみ)
- **交易関連**
  - コモディティの売却回数
  - 売却したコモディティの総数
  - 交易での損益
  - ROI(投資利益率)
  - 1トンあたりの利益
  - 1時間あたりの利益
  - 直近10取引の利益の推移
- **マテリアル関連:**
  - カテゴリ別マテリアル収集数集計
  - 収集したマテリアル名と個数の一覧
- **探索関連:**
  - 総ジャンプ回数
  - 総ジャンプ距離
  - 総スキャン数
  - スキャンデータの推定価値
  - 高価値スキャン数
  - 第一発見数
  - 直近60分の燃料残量の推移
  - スキャンインジケータ
    - FSSで星系内にELW, WW, AW, Terraformableを発見した場合に点灯
- **進行状況:**
  - 連邦/帝国ランク進行状況 (ログイン時、昇進時のみ)
  - 戦闘、交易、探索、CQCランク進行状況 (ログイン時、昇進時のみ)
  - 連邦/帝国/独立のミッション完了数
  - 旅客・救出ミッションの追跡：
    - 現在乗船中の総乗客数 (Passengers on Board)
    - 輸送完了した累計乗客数 (Passengers Transported)
    - 難民救出ミッション進行中のUIラベル自動切り替え (Passengers on Board -> Souls on Board) およびハイライト強調表示

## 🚀 導入手順

CUI操作を極力排除し、リリースビルドから動かす手順です。

### ステップ1: リリースのダウンロード
1. 本リポジトリの右側にある「Releases」から最新の `ed-dashboard.zip` をダウンロードします。
2. ダウンロードしたZipファイルを、任意のフォルダに解凍します。

### ステップ2: 設定ファイル（.env）の準備
1. 解凍したフォルダ内にある `.env.example` ファイルをコピーし、ファイル名を `.env` に変更します。
2. メモ帳などのテキストエディタで `.env` を開き、以下の項目を設定します：
   - `OBS_WEBSOCKET_PASSWORD`: OBSのWebSocketにパスワードを設定している場合はそのパスワードを指定します。
   - `JOURNAL_PATH`: Elite: Dangerous のジャーナルファイルが保存されているフォルダのパス。
     - デフォルト（Windows）の保存先: `C:\Users\<ユーザー名>\Saved Games\Frontier Developments\Elite Dangerous`
     - 保存先がデフォルトのままであれば、この設定は省略（空欄またはコメントアウト）しても自動的に検出されます。

### ステップ3: アプリの起動
1. フォルダ内にある `start-server.ps1` を右クリックし、**「PowerShellで実行」** を選択します。
   > 💡 **Node.js が未インストールの場合**  
   > 起動時に Node.js (および npm) のインストールを確認するメッセージが表示されます。「`Y`」を入力すると Windows Package Manager (`winget`) が起動し、Node.js LTS のインストーラーが起動します。画面の指示に従ってインストールを完了してください。  
   > インストール完了後は、環境変数を反映するために **一度ウィンドウを閉じ、再度 `start-server.ps1` を実行し直す** 必要があります。
2. 必要な依存ライブラリが自動的にインストールされた後、サーバーが起動します。
   > 💡 **PowerShellの実行ポリシーエラーが表示される場合**  
   > もし実行ポリシーの制限によりスクリプトが起動しない場合は、Node.jsをインストール後にコマンドプロンプトやPowerShellを開いて以下のコマンドを実行してください：
   > ```bash
   > npm install
   > npm start
   > ```
3. サーバーが起動すると、ブラウザでダッシュボード（`http://localhost:3000`）を開くことができるようになります。

### ステップ4: OBSとの連携（推奨設定）
本ダッシュボードはOBSの「ブラウザソース」としてオーバーレイ表示するのに最適化されています。以下の手順で設定すると、最も使いやすく配置できます。

1. OBSを起動し、ソース一覧の「＋」アイコンから **「ブラウザ」** を追加します。
2. プロパティで以下の通り設定します：
   - **URL**: `http://localhost:3000`
   - **幅**: `1050`
   - **高さ**: `1520`
3. 追加したブラウザソースを選択し、右クリック ➔ **「フィルタ」** を選択します。
4. フィルタウィンドウの左下「＋」アイコンから **「クロップ/パッド」** を追加します。
5. フィルタの設定で以下のように入力します：
   - **「相対的」にチェックを入れる**
   - **左**: `520` px
6. これにより、余分なスペースがクロップされ、**1列にすっきりとまとまったレイアウト**になり、ゲーム画面の横などに配置しやすくなります。
7. OBSのメニューから **[ツール] > [OBS WebSocket 設定]** を開き、**[WebSocketサーバーを有効にする]** にチェックが入っていることを確認してください。これにより、ダッシュボード上で録画や配信ステータスに連動したイベントログが正しく記録されます。

### ステップ5: スマートフォンやタブレットを「手元のリモコン」にする
本ダッシュボードは、ドラッグ＆ドロップによるカードの並び替えレイアウトがリアルタイムで他のブラウザへ同期されます。
- PCと同じローカルネットワーク（Wi-Fi等）に接続されたスマートフォンやタブレットから、PCのIPアドレス（例: `http://192.168.x.x:3000`）にアクセスします。
- または、実行しているPCで別のブラウザから`http://localhost:3000`を開きます。
- ブラウザの画面からカードを動かすと、OBSに表示されているダッシュボードのレイアウトも瞬時に同期して並び替わります。OBSを操作すること無く別のウィンドウを「サブディスプレイ兼リモコン」として活用できます。

## 🛠 開発者向け環境構築 (For Developers)

リポジトリをクローンしてカスタマイズやテストを行う開発者向けの情報です。

### 1. クローンと起動
1. リポジトリをクローンします：
   ```bash
   git clone https://github.com/d11j/ed-dashboard.git
   cd ed-dashboard
   ```
2. 依存パッケージをインストールします：
   ```bash
   npm install
   ```
3. サーバーを起動します：
   ```bash
   npm start
   ```

### 2. フォルダ構成
主要なファイルとフォルダの役割は以下の通りです：
- `server.js`: バックエンドサーバーのメインエントリーポイント
- `src/`: バックエンドの処理ロジック
  - `journalProcessor.js`: Elite: Dangerous のジャーナルファイル（JSON）の解析と統計処理
  - `utils.js` / `constants.js`: 共通のユーティリティおよび定数定義
- `public/`: フロントエンドのUIリソース
  - `index.html`: ダッシュボードのHTML構造
  - `assets/style.css`: UIのスタイル定義
  - `assets/main.js` / `uiUpdater.js`: UIの初期化と状態更新ロジック
  - `assets/websocketClient.js`: サーバーとのWebSocket双方向通信処理
  - `assets/charts.js`: 統計グラフの描画ユーティリティ
- `bin/simulator.js`: 開発用ジャーナルシミュレータスクリプト
- `tests/`: Vitestによる自動テストコードおよびテストデータ

### 3. 開発用ツールの実行

#### ジャーナルシミュレータの起動
ゲームを起動していなくても、テストログファイルを元にしてダッシュボードの動作を確認できるシミュレータです。
```bash
npm run simulate
```
このスクリプトを実行すると、テスト用のログデータ（`tests/fixtures/journal-test.log`）を読み込み、1秒ごとに最新のタイムスタンプに書き換えてジャーナルフォルダにダッシュボード用ダミーログを追記します。

#### テストの実行
本プロジェクトでは Vitest を用いた単体テスト（Unit Test）と結合テスト（Integration Test）が実装されています。
```bash
# すべてのテストを実行
npm test

# カバレッジレポートを出力してテストを実行
npm run test:coverage
```

#### コードスタイルの検証 (Linter)
ESLintによる静的解析チェックおよび自動修正が可能です。
```bash
# ルールチェック
npm run lint

# 自動修正
npm run lint:fix
```

## セッション再開と永続化 (Session Resume & Persistence)

- **セッションデータの保存**:
  サーバーシャットダウン時（Ctrl+C 等による終了時）や、ダッシュボード上の「RESET STATS」実行時に、その時点の統計データ（日時、各種合計値など）が `db.json` に自動保存されます。
- **セッションの再開 (RESUME SESSION)**:
  日付を跨いだり、サーバーを再起動した場合、通常は統計がリセットされますが、再起動後にゲームプレイを開始（ジャンプ、戦闘などのイベントの発生）前であれば、ダッシュボード上の「RESUME SESSION」ボタンを押すことで前回のセッションの続きから再開できます。
- **レイアウトの自動保存**:
  ダッシュボード上の各カードの並び順（ドラッグ＆ドロップで変更可能）は自動的に `db.json` に保存され、次回起動時や他のブラウザから接続した際も同じ並び順が適用されます。
