# Elite: Dangerous リアルタイムダッシュボード

## 概要
このアプリケーションは、戦闘、交易、探索、およびマテリアル収集などの統計情報をリアルタイムで表示するウェブベースのウィジェットです。
また、OBSと連携して録画開始時を起点としてタイムスタンプをイベントログとして記録する機能を持ちます。
各統計カードはドラッグ＆ドロップによって自由に並び替えることができ、レイアウトはリアルタイムで他のブラウザへ同期され、永続化されます。
さらに、主要な統計データ（賞金、利益、燃料残量）はグラフで視覚的に表示されます。

## 仕様
コンセプトは「今日の成果」であり、起動時当日のジャーナル及び、起動中に増えたジャーナルを収集対象とします。

日付を跨いだ場合も収集を継続して集計に加算し続けます。日付を跨いだあとに再起動した場合、前日の統計は集計対象外となるので統計がリセットされます。
ただし、ゲームのプレイイベント（ジャンプ、戦闘、取引など）がまだ発生していない場合に限り、ダッシュボード上の「RESUME SESSION」ボタンから直前のセッション状態を復元して再開することが可能です。

### 収集する統計情報
現在、以下の統計情報が収集・表示されます。

- **戦闘関連:**
  - 戦闘報酬の集計 (以下の総計)
    - 賞金額 (Bounty)
    - 戦闘貢献証券 (Combat Bond)
    - 1時間あたりの獲得賞金
    - 直近10キルの獲得賞金推移
  - ランク別撃墜数集計 (賞金首のみ)
  - 機種別撃墜数集計 (賞金首のみ)
- **交易関連**
  - コモディティの売却回数
  - 売却したコモディティの総数
  - 交易での損益
  - ROI(投資利益率)
  - 1トンあたりの利益
  - 1時間あたりの利益
  - 直近10取引の利益の推移
- **マテリアル関連:**
  - カテゴリ別マテリアル収集数集計
  - 収集したマテリアル名と個数の一覧
- **探索関連:**
  - 総ジャンプ回数
  - 総ジャンプ距離
  - 総スキャン数
  - スキャンデータの推定価値
  - 高価値スキャン数
  - 第一発見数
  - 直近60分の燃料残量の推移
  - スキャンインジケータ
    - FSSで星系内にELW, WW, AW, Terraformableを発見した場合に点灯
- **進行状況:**
  - 連邦/帝国ランク進行状況 (ログイン時、昇進時のみ)
  - 戦闘、交易、探索、CQCランク進行状況 (ログイン時、昇進時のみ)
  - 連邦/帝国/独立のミッション完了数
  - 旅客・救出ミッションの追跡：
    - 現在乗船中の総乗客数 (Passengers on Board)
    - 輸送完了した累計乗客数 (Passengers Transported)
    - 難民救出ミッション進行中のUIラベル自動切り替え (Passengers on Board -> Souls on Board) およびハイライト強調表示


# セットアップ手順

1. **フォルダの作成:**
   任意の場所に `ed-dashboard` という名前の新しいフォルダを作成します。

2. **ファイルの構成:**
   作成したフォルダ内に、以下のファイルとフォルダを配置します。
   - `package.json` (プロジェクトの定義ファイル)
   - `server.js` (バックエンドサーバーのコード)
   - `src/journalProcessor.js` (ジャーナルの処理をするモジュール)
   - `src/utils.js` (ユーティリティ関数)
   - `src/constants.js` (定数の定義)
   - `public/index.html` (フロントエンドUIのコード)
   - `public/assets/style.css` (フロントエンドのスタイルシート)
   - `public/assets/main.js` (フロントエンドのJavaScript)
   - `public/assets/uiUpdater.js` (UIの更新をするモジュール)
   - `public/assets/websocketClient.js` (サーバとのWebSocket通信をするモジュール)
   - `public/assets/charts.js` (埋め込みグラフの描画ユーティリティ)
   - `bin/simulator.js` (開発用ジャーナルシミュレータ)
   - `tests/` (自動テストディレクトリ)
   - `eslint.config.js` (ESLint設定ファイル)
   - `vitest.config.js` (Vitest設定ファイル)

3. **ライブラリのインストール:**
   ターミナル（コマンドプロンプトやPowerShellなど）を開き、`ed-dashboard` フォルダに移動します。
   
   `cd path/to/ed-dashboard`

   そして、以下のコマンドを実行して、必要なライブラリをインストールします。

   `npm install`

4.  **OBSのWebSocket設定:**
    OBSのメニューから **[ツール] > [OBS WebSocket 設定]** を開き、**[WebSocketサーバーを有効にする]** にチェックを入れてください。パスワードを設定している場合は、そのパスワードを次のステップで利用します。

5.  **環境変数の設定:**
    プロジェクトのルートにある `.env.example` ファイルをコピーし、ファイル名を `.env` に変更します。
    - `OBS_WEBSOCKET_PASSWORD`: OBSのWebSocketにパスワードを設定している場合に指定します。
    - `JOURNAL_PATH`: Elite: Dangerousのジャーナルファイルが保存されているディレクトリのパス。デフォルト以外の場所にインストールしている場合に指定が必要です。

6.  **アプリケーションの起動:**
    ターミナルで `npm start` コマンドを実行するだけで、サーバーが起動します。
    ```bash
    npm start
    ```
    または、PowerShellスクリプトを実行します。
    ```powershell
    ./start-server.ps1
    ```

7. **ダッシュボードの表示:**
   ウェブブラウザを開き、アドレスバーに以下のURLを入力します。
   http://localhost:3000

   ブラウザにダッシュボードが表示され、ゲームをプレイするとリアルタイムで数値が更新されます。


# セッション再開と永続化 (Session Resume & Persistence)

- **セッションデータの保存**:
  サーバーシャットダウン時（Ctrl+C 等による終了時）や、ダッシュボード上の「RESET STATS」実行時に、その時点の統計データ（日時、各種合計値など）が `db.json` に自動保存されます。
- **セッションの再開 (RESUME SESSION)**:
  日付を跨いだり、サーバーを再起動した場合、通常は統計がリセットされますが、再起動後にゲームプレイを開始（ジャンプ、戦闘などのイベントの発生）する前であれば、ダッシュボード上の「RESUME SESSION」ボタンを押すことで前回のセッションの続きから再開できます。
- **レイアウトの自動保存**:
  ダッシュボード上の各カードの並び順（ドラッグ＆ドロップで変更可能）は自動的に `db.json` に保存され、次回起動時や他のブラウザから接続した際も同じ並び順が適用されます。


# 開発とテスト (Development & Testing)

開発およびテスト用途として、以下のコマンドやスクリプトが用意されています。

### 1. ジャーナルシミュレータの起動
ゲームを起動していなくても、テストログファイルを元にしてダッシュボードの動きを確認できるシミュレータスクリプトです。
```bash
npm run simulate
```
このスクリプトを実行すると、実機ログデータ（`tests/fixtures/journal-test.log`）を読み込み、1秒ごとに最新のタイムスタンプに書き換えてジャーナルフォルダに追記します。

### 2. テストの実行
本プロジェクトでは Vitest を用いた単体テスト（Unit Test）と結合テスト（Integration Test）が実装されています。
```bash
# すべてのテストを実行
npm test

# カバレッジレポートを出力してテストを実行
npm run test:coverage
```

### 3. コードスタイルの検証 (Linter)
ESLintによる静的解析チェックが可能です。
```bash
# ルールチェック
npm run lint

# ルールチェックと自動修正
npm run lint:fix
```
